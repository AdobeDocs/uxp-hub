# UXP for Newbies

If you've never written JavaScript, HTML, or CSS, but you want to develop plugins using UXP, you're in for a bit of a learning curve. It's not a steep one, and plenty of people have climbed it before you. Dive in, and come back to these pages when you're ready.

A couple of things before you start:

You'll need to know a bit about your host application. If that's Photoshop, a passing knowledge of layers, selections, and basic tools (such as the pen and type tools) is a good start.

Next, learn some basic JavaScript. There's an overwhelming amount of material out there, so start with a focused resource rather than trying to read everything. One popular starting point is the [Mozilla Developer Network tutorials](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Scripting). Different people learn differently: some prefer documentation, some prefer video, some prefer diving in and making mistakes. Find what works for you.

The same applies to learning HTML and CSS, though both are generally easier than JavaScript to pick up. You don't need to go deep on any of these: the HTML, CSS, and JavaScript required for UXP is simpler than what a typical website needs.

Once you're ready to start coding, looking at sample code and adapting it for your own projects is a completely normal way to learn, as long as you take the time to understand what it's doing rather than just copying it. To get started, [look here](https://github.com/AdobeDocs/uxp-photoshop-plugin-samples) for sample code.

Finally, ask questions. There are many helpful people on the [Adobe Creative Cloud Developer Forums](https://forums.creativeclouddeveloper.com/), and chances are you'll find answers there.
